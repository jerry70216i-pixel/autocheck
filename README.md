# 工程照片查驗表自動化系統

目前實作 V1：在 V0 的照片選取、預覽、排序、移除與匯出流程上，新增工項選擇與查驗說明範本帶入。

## 啟動方式

不需要安裝套件。請在專案根目錄執行：

```powershell
python -m http.server 8000 -d public
```

若系統沒有 `python`，可使用 Codex 內建 Python：

```powershell
C:\Users\user\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe -m http.server 8000 --bind 127.0.0.1 -d public
```

然後開啟：

```text
http://127.0.0.1:8000
```

## V1 使用方式

1. 填寫工程名稱、日期、位置、查驗項目。
2. 選取多張工程照片。
3. 在照片預覽區檢查順序。
4. 視需要調整照片順序或移除照片。
5. 每張照片可選擇工項。
6. 選擇工項後，若查驗說明仍是空白，系統會帶入該工項的預設說明。
7. 使用者可以手動修改每張照片的查驗說明。
8. 點選「重新產生表格預覽」確認固定表格。
9. 點選「匯出 HTML 表格」下載含照片與說明的表格。

## V1 向後相容規則

- 不選工項也可以匯出。
- 不填查驗說明也可以匯出。
- 工項與說明不得成為照片預覽或匯出的必要條件。
- IndexedDB 失敗時，基本照片選取、預覽與匯出流程仍應可用。

## 本機資料儲存

- 工項與查驗說明範本目前儲存在瀏覽器本機 IndexedDB。
- 資料儲存邏輯封裝在 `public/storage-repository.js`。
- UI 不直接操作 IndexedDB。
- 目前不會把照片 binary 或 data URL 存入 IndexedDB。

## V1 限制

- 不做 AI。
- 不做 RAG。
- 不做 AI Agent。
- 不做帳號登入。
- 不做區域網路部署。
- 不做後端資料庫。
- 不做多格式輸出。
- 不做多表格模板。

## 手動驗收重點

1. 可以選取或上傳多張照片。
2. 可以顯示照片預覽。
3. 照片順序正常。
4. 可以移除照片。
5. 每張照片旁邊有工項下拉選單。
6. 每張照片旁邊有查驗說明欄位。
7. 選擇工項後會自動帶入預設說明。
8. 手動修改說明後，再切換工項不會覆蓋既有說明。
9. 不選工項也可以匯出。
10. 不填說明也可以匯出。
11. 匯出檔案可以正常開啟。
12. 匯出表格內照片正確。
13. 匯出表格內說明正確。
14. 重新整理頁面後，工項與說明資料仍保留。
