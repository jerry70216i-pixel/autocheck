# 工程照片查驗表自動化系統開發計畫

## 目前狀態

V0 已完成並恢復穩定。現在準備以向後相容方式推進 V1。

重要原則：V1 是 V0 的擴充，不是取代 V0。任何 V1 修改都必須通過 `docs/V0_REGRESSION_CHECKLIST.md`。

## V0：已完成

### 已完成功能

- 使用者可以選取或上傳照片。
- 系統可以顯示照片預覽。
- 使用者可以手動輸入或修改查驗說明。
- 使用者可以調整照片順序。
- 使用者可以移除照片。
- 使用者可以匯出 HTML 表格。
- 照片可以正確放入表格。

### V0 不做事項

- 不做 AI。
- 不做 RAG。
- 不做 AI Agent。
- 不做帳號權限。
- 不做資料庫。
- 不做後端 API。
- 不做多格式輸出。
- 不做多表格模板。

### V0 保護條件

- 工項與說明不得成為 V0 匯出的必要條件。
- 沒有選工項也可以匯出。
- 沒有查驗說明也可以匯出。
- 本機資料儲存失敗時，基本照片匯出流程不應崩潰。

## V1：工項與查驗說明範本

### V1 目標

- 建立工項資料。
- 建立查驗說明範本資料。
- 每張照片可以選擇工項。
- 選擇工項後，自動帶入對應的預設查驗說明。
- 使用者可以手動修改每張照片的查驗說明。
- 匯出表格時，除了照片，也能輸出對應說明。
- 資料先採用瀏覽器本機儲存，優先使用 IndexedDB；若目前架構不適合，再提出 localStorage 方案。
- 資料儲存邏輯必須封裝，不可直接散落在 UI 元件中。

### V1 不做事項

- 不做 AI 圖像辨識。
- 不做 RAG。
- 不做 AI Agent。
- 不做帳號權限。
- 不做區域網路部署。
- 不做後端資料庫。
- 不做多格式輸出。
- 不重寫整個專案。

### V1 完成條件

- V0 regression checklist 全部通過。
- V1 checklist 全部通過。
- 可以建立預設工項資料。
- 可以建立預設查驗說明範本。
- 每張照片可以選擇工項。
- 選擇工項後可以自動帶入預設說明。
- 使用者可以修改說明。
- 匯出時可以輸出照片與說明。
- 未填工項或說明時仍可匯出。
- 重新整理頁面後，工項與說明資料仍保留。
- 資料儲存邏輯集中在 repository 或 storage 層。
- UI 不直接操作 IndexedDB 或 localStorage。

### V1 預計修改檔案

- `AGENTS.md`

- `docs/V0_REGRESSION_CHECKLIST.md`
- `docs/V1_CHECKLIST.md`

- `public/storage-repository.js`
